# Portfolio Review & Reflection

## Part A – Review and Improvements

### 1. Improvements that I Made

After reviewing my portfolio code, I made the following improvements:

1. Added semantic HTML elements such as `header`, `nav`, `main`, `section`, and `footer` to improve structure and readability.
2. Separated HTML, CSS, and JavaScript into different files for better maintainability.
3. Improved JavaScript safety by checking if the button exists before adding an event listener.
4. Added dedicated **Skills** and **Projects** sections to make the portfolio more realistic and professional.
5. Improved layout spacing and overall structure.

## Part B – Ethics and Risks

### 1. Overreliance on AI

Risk: Developers may become too dependent on AI and use code without fully understanding it.

How I will mitigate it:
I will always review and rewrite AI-generated code to ensure I understand how it works before using it.

### 2. Security Risks

Risk: AI may generate insecure code or unsafe practices.

Mitigation: I will review AI-generated code carefully and check official documentation when needed.

### 3. Incorrect Information

Risk: AI can sometimes provide outdated or incorrect information.

Mitigation: I will verify important technical details using trusted sources like official documentation.
